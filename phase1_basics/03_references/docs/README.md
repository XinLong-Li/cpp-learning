# 引用（References）

## 引用是什么？

**引用是变量的别名**——它不是指针，不是对象，就是同一个东西的另一个名字。

```cpp
int x = 42;
int& r = x;   // r 是 x 的引用（别名）
r = 100;      // 修改 r 就是修改 x
std::cout << x; // 输出 100
```

---

## 左值引用 vs 右值引用

| 引用类型 | 语法 | 绑定对象 | 主要用途 |
|----------|------|---------|---------|
| 左值引用 | `T&` | 左值（有名字/地址的变量） | 函数参数、避免拷贝 |
| const 左值引用 | `const T&` | 左值**和**右值 | 函数参数（万能只读引用） |
| 右值引用 | `T&&` | **仅**右值（临时对象、字面量） | 移动语义 |

```cpp
int x = 5;
int& lr = x;         // ✅ 左值引用绑定左值
int& lr2 = 5;        // ❌ 左值引用不能绑定右值
const int& cr = 5;   // ✅ const 左值引用绑定右值（生命周期延长）
int&& rr = 5;        // ✅ 右值引用绑定右值
int&& rr2 = x;       // ❌ 右值引用不能绑定左值
```

---

## 引用的核心特性

1. **引用必须初始化**（没有"空引用"）
   ```cpp
   int& r; // ❌ 编译错误！
   ```

2. **引用不可重绑定**（一旦绑定，终身绑定）
   ```cpp
   int a = 1, b = 2;
   int& r = a;
   r = b;   // 这是把 b 的值赋给 a，不是让 r 引用 b
   ```

3. **引用不是对象**，所以没有"引用的引用"、"引用的指针"、"引用的数组"

4. **引用作为参数传递时不拷贝**（这是最重要的用途）
   ```cpp
   void process(const std::string& s); // 不拷贝，高效
   ```

---

## 什么时候用什么传递？

| 情形 | 参数写法 | 原因 |
|------|---------|------|
| 只读，小类型（如同 int） | `T x` | 值传递可能更快（寄存器） |
| 只读，大类型（如 string） | `const T& x` | 避免拷贝 |
| 修改参数原值 | `T& x` | 需要修改 |
| 接管所有权（移动） | `T&& x` 或 `T x`+`std::move` | 移动语义 |
| 转发参数 | `T&& x`（万能引用） | 完美转发 |

## 常见陷阱

1. **悬垂引用（Dangling Reference）**
   ```cpp
   int& foo() {
       int x = 42;
       return x;  // ❌ 返回局部变量的引用！x 已销毁
   }
   
   const int& r = std::string("hello"); // ❌ string 临时对象已销毁！
   const std::string& s = "hello";      // ✅ const 引用直接绑定到临时对象，生命周期延长
   ```

2. **引用的引用？不，引用折叠**
   ```cpp
   using lref = int&;
   lref& r = x;  // int& & → int&（引用折叠成左值引用）
   using rref = int&&;
   rref& r2 = x; // int&& & → int&（左值引用"赢"了）
   ```
